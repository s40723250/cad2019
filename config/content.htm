製圖應用程序的一些有用功能

1）選擇第一個視圖後，可以添加其他正交視圖，並通過單擊幾個按鈕將其對齊。

2）每個視圖都直接與實體關聯。 因此，當更改實體時，工程圖將隨視圖和尺寸一起直接更新。

3）草稿註釋（尺寸，標籤和帶引線的符號）直接放置在工程圖上，並在更改實體時自動更新。

創建草稿

➢打開文件Arborpress_rack.prt

➢從NX 12界面中，選擇File→Draft如圖所示，或選擇Application選項卡並選擇Drafting

首次打開“繪圖應用程序”時，會彈出一個窗口，要求輸入諸如模板，標準尺寸或自定義尺寸，單位和投影角度。

尺寸設定

圖紙大小可選擇已設定好的尺寸或自行定義尺寸。

單位設定

單位遵循3-D模型的默認單位。如果從“製圖應用程序”開始，則需要在此處選擇單位。

投影面設定

可以選擇“第一角法”或“第三角法”

* 要開始使用“繪圖應用程序”，從創建“標準尺寸”圖紙開始：

➢單擊標準尺寸單選按鈕

➢在尺寸窗口的下拉菜單中，選擇尺寸為11 x 17的圖紙B。

➢使用下拉菜單並在“比例”下選擇“自定義比例”，將比例更改為1:25。

➢點擊確認

➢打開“製圖應用程序”

➢開起草圖應用程序界面(會彈出一個對話框，它將幫忙你選擇零件或視圖和其他選項)

➢更改選項和視圖，然後單擊完成

➢選擇插入→視圖→基礎，或在視圖組中單擊基礎視圖

(具有“視圖”和“比例”選項的“基本視圖”對話框將隨對象的浮動圖形一起顯示)

➢選擇前面的視圖

在屏幕上找到“前視圖”投影(可以在屏幕上移動鼠標光標，然後單擊要查看的位置。)

➢設置“前視圖”後，將彈出另一個對話框，要求您在“圖紙邊界”內屏幕上的任何位置設置其他視圖。

通過在第一個視圖上移動光標來找到不同的視圖( 如果要在關閉此文件或更改為其他命令模式後添加任何正交視圖)

➢選擇插入→視圖→投影視圖或從“視圖”組中選擇“投影視圖”圖標

➢創建所有其他正交投影視圖，然後在屏幕上的所需位置單擊。

如果關閉了“投影視圖”對話框，則可以通過單擊視圖組中的“投影視圖”圖標重新將打開它 

➢移動光標並單擊以獲取其他視圖

➢單擊“投影視圖”對話框上的“關閉”或按鍵盤上的<Esc>鍵以關閉窗口

在創建尺寸之前，刪除每個視圖中的邊框(這會增加與實體線的混淆)

➢選擇菜單→首選項→起草，或單擊快速訪問工具欄中的圖標以找到起草首選項

“製圖首選項”窗口將彈出。

➢單擊查看選項卡按鈕

5.5產品和製造信息

產品和製造信息（PMI）是NX中的重要應用程序之一，它提供用於在3D環境中記錄產品的註釋工具。 PMI應用程序包括一個全面的3D註釋環境，允許設計團隊將詳細信息（例如幾何尺寸和公差（GD＆T），表面光潔度，焊接信息，材料規格，註釋，政府安全信息或專有信息等）共享給3D模型。 。 PMI符合3D產品定義的行業標準，因此從事協作項目的產品團隊將使用3D模型作為完全記錄產品和製造信息的合法方法。

如何打開零件文件，在PMI應用程序中的3D模型上創建尺寸和註釋，並學習如何將尺寸和註釋繼承到Drafting應用程序。

➢打開文件Impeller_impeller.prt

➢從NX 12界面中選擇File→PMI（打開復選標記）

這應該在“工具”和“應用程序”選項卡之間創建一個附加的選項卡PMI。 選擇“ PMI”選項卡以進入如下所示的PMI應用程序。

 

功能區欄將具有“尺寸”，“註釋”，“自定義符號”，“補充幾何”，“特殊”和“安全標記”組。 每個小組都有幾個選項，可以幫助描述建模的3D零件。 例如:尺寸/表面粗糙度/註釋中的標註的尺寸選項。

➢單擊快速圖標

➢選擇葉輪的端面作為第一對象和第二對象，以插入線性尺寸或單擊“線性”圖標執行相同的任務

➢單擊“註釋”組中的“註釋”圖標以提供任何註釋，或單擊“表面粗糙度”圖標，選擇對象，文本位置和引出線以插入特定的表面粗糙度詳細信息（如果需要）

在確定PMI尺寸後，葉輪的Trimetric視圖如下所示

About

此內容管理系統以 https://github.com/mdecourse/cmsimde 作為 submodule 運作, 可以選定對應的版本運作, cmsimde 可以持續改版, 不會影響之前設為 submodule, 使用舊版 cmsimde 模組的內容管理相關運作.

利用 cmsimde 建立靜態網誌方法:

1. 在 github 建立倉儲, git clone 到近端

2. 參考 https://github.com/mdecourse/newcms, 加入除了 cmsimde 目錄外的所有內容

以 git submodule add https://github.com/mdecourse/cmsimde cmsimde

建立 cmsimde 目錄, 並從 github 取下子模組內容.

3.在近端維護時, 更換目錄到倉儲中的 cmsimde, 以 python wsgi.py 啟動近端網際伺服器.

動態內容編輯完成後, 以 generate_pages 轉為靜態內容, 以 git add commit 及 push 將內容推到遠端.

4. 之後若要以 git clone 取下包含 submodule 的所有內容, 執行:

git clone --recurse-submodules https://github.com/mdecourse/newcms.git

Develop

https://github.com/mdecourse/cmsimde 的開發, 可以在一個目錄中放入 cmsimde, 然後將 up_dir 中的內容放到與 cmsimde 目錄同位階的地方, 使用 command 進入 cmsimde 目錄, 執行 python wsgi.py, 就可以啟動, 以瀏覽器 https://localhost:9443 就可以連接, 以 admin 作為管理者密碼, 就可以登入維護內容.

cmsimde 的開發採用 Leo Editor, 開啟 cmsimde 目錄中的 cmsimde.leo 就可以進行程式修改, 結束後, 若要保留網際內容, 只要將 cmsimde 外部的內容倒回 up_dir 目錄中即可後續對 cmsimde 遠端倉儲進行改版.

init.py 位於  up_dir 目錄, 可以設定 site_title 與 uwsgi 等變數.
